Multi-GPU Vector Addition

Compute the element-wise sum of two vectors A and B into output vector C on 2 GPUs.

Your solve() function is invoked once on each rank. The same input tensors A and B are available on every rank (fully replicated), and each rank must produce the full result in its own C. You may parallelize the work across ranks and use collectives (PyTorch: torch.distributed; CUDA: NCCL with the rank/world-size and NCCL unique-id file exposed via environment variables) to combine partial results.

Environment variables available inside solve():

  • RANK: this rank's index (0-based)
  • WORLD_SIZE: total number of ranks (2 for this challenge)
  • LOCAL_RANK: local device index for this rank
  • LEETGPU_NCCL_ID_FILE: (CUDA only) path to a shared file for bootstrapping NCCL via ncclGetUniqueId / ncclCommInitRank

For PyTorch solutions the process group (nccl backend) is pre-initialized by the runner — just call torch.distributed.all_reduce, torch.distributed.all_gather, etc.

Constraints

  • 1 ≤ N ≤ 107
  • A, B, C are contiguous float32 tensors on the GPU

This is a Pro-only multi-GPU challenge.
