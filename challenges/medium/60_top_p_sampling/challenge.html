Write a GPU program that computes the top-p (nucleus) probability distribution for LLM inference.

Top-p sampling is a text generation technique where you identify the smallest set of tokens whose cumulative probability exceeds threshold p, then renormalize their probabilities. This balances randomness and quality better than pure top-k or greedy sampling.

Given logits (unnormalized scores) from a language model:

  1. Convert logits to probabilities using softmax
  2. Sort tokens by probability (descending)
  3. Find the smallest set where cumulative probability ≥ p (the "nucleus")
  4. Renormalize the nucleus probabilities to sum to 1
  5. Output top_p_probs[i] = renormalized probability if token i is in the nucleus, 0 otherwise

Implementation Requirements

  • Use only native features (external libraries are not permitted)
  • The solve function signature must remain unchanged
  • Ensure numerical stability when computing softmax

Example 1:

Input:
  logits = [1.0, 2.0, 3.0, 0.5]
  p = 0.9

Output:
  top_p_probs = [0.0, 0.2689, 0.7311, 0.0]
  (tokens 1 and 2 form the nucleus; their probabilities are renormalized to sum to 1)

Example 2:

Input:
  logits = [10.0, 1.0, 1.0]
  p = 0.5

Output:
  top_p_probs = [1.0, 0.0, 0.0]
  (token 0 alone exceeds p; renormalized probability is 1.0)

Constraints

  • 3 ≤ vocab_size ≤ 50,000
  • -100.0 ≤ logits[i] ≤ 100.0
  • 0.0 < p ≤ 1.0
  • top_p_probs values sum to 1.0 for tokens in the nucleus, 0.0 elsewhere
  • Performance is measured with vocab_size = 50,000
